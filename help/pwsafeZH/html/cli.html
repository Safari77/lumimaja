命令行参数

调用 Password Safe 不带参数将导致应用程序提示为最后被打开过的数据库输入密码或在计算机未打开有过早期数据库时提示输入新数据库密码(例如: 首次使用 Password Safe)。 不管如何，尽可能修改 Password Safe 的这些参数引用如下:

  • pwsafe 数据库名称
    该参数将打开指定的数据库文件，而不打开最后一次被打开过的数据库。 假如只考虑一个文件名而没有路径，它将搜索其中被引用的应用程序目录。 注意，如果文件名和/或为空，将被双引号标识。
  • pwsafe -r [数据库名称]
    该参数将以只读模式打开指定的数据库。如果未指定数据库，则应用程序将提示用户指定一个需要被打开为只读模式的数据库。
  • pwsafe -v 数据库名称
    该参数将验证数据库在输入正确的密码语句后，校正内部不一致的错误。 对于通过其它应用程序写入数据库时这是有帮助的。
  • pwsafe -e 文件名
    该参数将提示用户给予一个密码语句和使用一个密码语句密匙加密文件。 注意: 文件可以是任何文件。被加密的文件将与原始文件同样的名称 ，是具有附加后缀为 ".PSF" 的文件。
  • pwsafe -d 文件名
    该参数将提示用户给予一个密码语句和使用一个密码语句密码解密文件。 注意: 该参数将只作用于被加密的文件已引用了 pwsafe '-e' 选项时(查看前一个参数)。
  • pwsafe -c
    该参数将启动应用程序的关闭菜单功能，即程序启动后没有数据库和没有初始打开对话(要访问数据库，使用文件菜单)。
  • pwsafe -s
    该参数将应用程序启动为“静默”，即最小化和没有数据库。 当解除应用程序最小化时，用户可见打开对话框(该选项表示启动应用程序时为登录界面，通过一个快捷方式启动用户文件夹)。 注意: 该参数调用设置应用程序在系统托盘。
  • pwsafe -m
    该参数与 '-c' 选项等同，具有增添应用程序启动时为最小化。
另外，下列选项承认:
  • -u 用户名
    该参数将导致应用程序读取和写入在指定 用户名 下的参数选择，代替登录名称。
  • -h 主机名称
    该参数将导致应用程序读取和写入在指定主机名称下的参数选择，代替本计算机名称。
  • -g 配置_文件
    该参数将导致指定用于加载和保存参数选择的文件，代替默认的 pwsafe.cfg。如果只给予一个文件名称而没有一个路径， 程序将搜索调用应用程序目录。注意如果文件名称 和/或 路径为空，将被双引号标识。
这些参数可能有助于假设你希望共享相同参数选择跨越多个计算机时，例如: 当运行 Password Safe 于 U 盘时。
