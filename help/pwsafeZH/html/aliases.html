别名密码

如果你使用的是一个“单点登录”的系统，则允许你共享一样的密码跨越不同的计算机/服务器/应用程序等等。 可以设置 Password Safe 相应的项目，因此，改变一个密码会影响所有其他相关的项目，这个便是被称为“别名”使用的项目。本节说明如何定义和使用别名。

概要

基本概念是允许一个项目的密码引用到另一个项目。 该“引用”项目被命名为基本和 “指向”项目(或多项目)被称为别名。 当一个项目设定到另一个别名项目，该别名密码实际上“跟随”项目的密码: 如果复制别名密码到剪贴板，得到的复制密码其实是基本项目的密码。 如果更改了基本密码，则立即反映到所有与基本项目关联的别名项目上。 虽然这些听起来有点复杂，但实际用起来其实很简单和很直观的。 最难的部份是设置别名，如下所述。

定义一个别名

在 Password Safe，一个项目可以参考另一项目的密码，通过一个特殊格式的密码访问项目。 格式为引用项目的“名称”使用方括号标识。 大多情况下，可以看作“名称”为一个项目标题字段的同义词，因此，如果基本项目标题是 “基本”，例如，输入“[主要]”(无引号)在另一项目密码字段足以定义另一项目为别名的“基本”。

在一般情况下，“名称”可以包含所有下列字段，使用冒号分隔: 组、标题和用户名。注意如果标题在数据库中是唯一的， 则其它字段为可选。同样，如果组和标题一致或标题与用户同时指定一个唯一项目， 则对应用户或组不必指定。

在 Password Safe 项目中，只有标题字段是强制性的。组和用户名字段为可选，只要“组/标题/用户 名称” 的组合结果是唯一的便可。如上表明，别名“密码”处于 [g:t:u] 而实际上你只有指定足够识别的唯一信息给基本项目。 因此，如果只有一个项目的数据库具有标题“t”，则 [t] 就足够了 - 因为标题时强制性的和只有一个项目的情况，这是假定的标题。 如果有一个以上的项目具有此标题，将被警告和应该需要更具体鉴别基本项目，通过添加它的任一个组或用户名或两者！ 只要有一个唯一的项目信息可提供，下列任何一种格式都可被接受: [g:t:u]、[g:t]、[t:u] 或 [t]。 指定一个冒号在没有一个值的空白字段，例如: [g:t:] 说明项目具有标题“t”与组“g”和不具有用户名的值以及 [:t:] 说明项目具有标题“t”在根目录无用户名的值等等。

下面的例子说明了这一点。

例子

1. 如果基本项目的标题是“LDAP 服务器”和没有另外的项目使用该标题， 则通过定义一个项目 "[LDAP 服务器]" 为别名在密码字段。

2. 如果有“LDAP 服务器”项目在两个不同的组，说明 “Dept. A”和“Dept. B”那么可以指定一个别名给后者“[Dept. A:LDAP 服务器]”。 注意，如果提供的信息能识别唯一的基本项目，可以不必指定用户名。

3. 不同的用户名也可以被用于不同的类似基本项目之间，例如，“[LDAP 服务器:赵]”与“[LDAP 服务器:孙]”。

4. 最后，所有三字段可指定，如:“[Dept. A:LDAP 服务器:赵]”

显示

基本项目，即项目至少有一个别名“指向”它们，在树状目录视图显示为一个绿色三角形而不是绿色正方形。
