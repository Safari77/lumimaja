设置一个密码策略

一个密码策略描述规则用于创建一个必需的密码字母，大小写字母、长度和任何特殊字符。 有些惯例只需要小写字符，而有些却要求大写字符、数字和使用特殊字符以及规定的最小长度。 使用密码允许最大长度和允许字符混合最大限度，总的来说这是一个好的建议。请根据它们的要求校验调整准确的密码。

密码策略对话允许用户设定全局密码生成策略，该策略可被临时忽略和在编辑/查看项目对话任何时间手工编辑密码。

选定密码规则

请为每个“使用”项目(例如: 使用数字，使用符号，等等。)在 Password Safe 对话框点击包括生成密码指定字符的复选框。 每个密码将被生成所指定的长度(不再缩短)。唯一例外的是仅使用十六进制数字复选框被勾选时，密码将只包括这些字符代表十六进制数字。选择该选项时，所有其它勾选框将被禁用。

选择生成可发音密码将促使 Password Safe 生成一个“无意义”但应该可以读音的英文字词，例如:“ingstria”，“ringsony等等。当用户需要一个密码具安全又相对容易记住和在没有 Password Safe 的情况时，这是非常有帮助的。选择该选项使用数字和/或使用符号将使一些字母替换成相等于“黑客语”特征的字符，例如:“E”可能替换成“3”、“S”替换成“5”等等。

当使用符号被选中时将附加这些字符: + - = _ @ # $ % ^ & ; : , . < > / ~ \ [ ] ( ) { } ? ! | 除非仅使用易解读字符也被选中，在这种情况下，只有字符: + - = _ @ # $ % ^ & < > / ~ \ ?

在仅使用易解读字符被选中时使用这些字母: a b c d e f g h i j k m n o p q r s t u v w x y z 和 A B C D E F G H J K L M N P Q R T U V W X Y 以及数字被限制为 3 4 6 7 8 9。
