Text Export & Import Format

Warnung: Der Export von sensiblen Informationen aus einer Passwort Safe Datenbank in einer Textdatei bringt erhebliche Sicherheitsprobleme mit sich. Bei diesem Vorgang wird eine ungeschützte Kopie der Passwörter in der Datenbank erstellt. Das Löschen dieser Kopie nach deren Verwendung ist nicht ausreichend und darum müssten andere Anwendungen eingesetzt werden, um die Daten "garantiert" zu löschen (z.B. externe Hilfsmitteln zum "Schreddern" der Daten). Der Export von sensiblen Informationen aus einer Passwort Safe Datenbank umgeht die Sicherheit der Anwendung.

Hinweis: Das Exportieren und Importieren zu/von Textdateien wurde in Version V3.06 geändert. Auf dieser Seite wird das aktuelle Format beschrieben.

Exportierter Text wird:

  • beginnen mit einer Titel-Zeile als erster Datensatz in der Textdatei
  • ein Trennzeichen anwenden, um das "neue Zeile"-Zeichen in Notizen mit mehr als einer Zeile und den Punkt ('.') im Feld Titel zu ersetzen. Das Standardtrennzeichen ist '»' [Alt+0187]. Es kann, wenn notwendig, überschrieben werden.

Importierter Text setzt voraus dass:

  • eine Titel-Zeile als erste Zeile in der Textdatei vorhanden ist.
  • ein Trennzeichen da ist, um das "neue Zeile"-Zeichen in Notizen mit mehr als einer Zeile und den Punkt ('.') im Feld Titel zurückzuersetzen. Das Standardtrennzeichen ist '»' [Alt+0187]. Es kann, wenn notwendig, überschrieben werden.

Die Titel-Zeile enthält für einen vollständigen Export standardmäßig folgende Felder, getrennt mit einem TAB-Zeichen.

  • Gruppe/Titel
  • Benutzername
  • Passwort
  • URL
  • Autom. Eingabe
  • Erstellungszeit
  • Passwort Änderungszeit
  • Letzter Zugriffszeit
  • Passwort Ablaufdatum
  • Passwort Ablaufintervall
  • Datensatz Änderungszeit
  • Passwort Richtlinie
  • Historie
  • Kommando Ausführen
  • DCA
  • E-Mail
  • Schutz Status
  • Symbolische Liste zur Passwortgenerierung
  • Notizen

Beim Exportieren von Daten, können Sie auf "Erweitert" drücken, um eine beliebige Anzahl dieser Felder auszuschließen in der Titel-Zeile und den Datensätzen. In der Option "Erweitert" (siehe unten) definieren Sie eine Untermenge an Einträge die Sie Exportieren wollen.

Beim Importieren muss die eingebaute Titel-Zeile exakt die Elemente als 'Spalten-Köpfe' enthalten, wie oben definiert. Wenn irgendein 'Spalten-Kopf' wiederholt wird, werden nur die Daten des letzten übernommen beim Importieren. Wenn ein 'Spalten-Kopf' nicht in der obigen Liste vorkommt, wird er beim Importieren nicht berücksichtigt. Die Daten müssen mit dem Trennzeichen unterteilt sein wie es im Dialog Text Importieren definiert wurde.

Hinweis:
  • Um überhaupt Daten zu Passwort Safe importieren zu können müssen unbedingt die Felder Titel und Passwort vorhanden sein. Sie dürfen weder nur Leerzeichen noch leer sein, wie es auch der Fall ist beim Einfügen eines Eintrages zu einer Datenbank.
  • Es besteht keine Einschränkung in der Reihenfolge der 'Spalten-Köpfe'. Passwort Safe wird die mitgebrachte Titel-Zeile verwenden, um die spezifizierten Daten im richtigen Feld abzulegen.

Zeit-Felder können eines der 5 folgende Formate haben:
1. yyyy/mm/dd hh:mm:ss z.B., 2007/01/26 19:31:25
2. ddd MMM dd hh:mm:ss yyyy z.B., Fri Jan 26 19:31:25 2007
3. yyyy-mm-ddThh:mm:ss z.B., 2007-01-26T19:31:25
4. yyyy-mm-ddThh:mm:ssZ (nur XML) z.B., 2007-01-26T19:31:25Z
5. yyyy-mm-ddThh:mm:ssthh:mm (nur XML) z.B., 2007-01-26T19:31:25+01:00
6. 0 das Feld unbesetzt lassen
7. now die Zeit des Importierens anwenden

wo:

yyyydas voll ausgeschriebene Jahr. Der Wert liegt zwischen 1970 und 2038.
mmder Monat; z.B. 01 = Januar, 02 = Februar usw.
ddder Tag im Monat
MMMdie Kurzform des Monats; z.B. Jan = Januar, Feb = Februar usw.
ddddie Kurzform des Wochentages; z.B. Mon = Montag, Tue = Dienstag (Tuesday) usw.
hhdie Stunde (24-teilig).
mmdie Minuten
ssdie Sekunden
Hinweis: Format '2' ist das Ergebnis der ANSI C Funktion 'asctime' und Format '3' ist konform zum ISO 8601 Standard.

Hinweis: 2

Die Passworthistorie wird im exportierten Datensatz in einem Textfeld abgebildet durch eine Kopf gefolgt von einer Anzahl Felder mit den alten Passwörtern:

Passworthistorie Kopf:

1 hexadezimales Zeichen Status zum Speicher der Passworthistorie für diesen Eintrag (0 = nein; 1 = ja)
2 hexadezimale Zeichen Maximum Anzahl an Einträgen für diesem Satz
2 hexadezimale Zeichen Anzahl der aktuel abgespeicherten Einträge

Jeder Eintrag der Passworthistorie:

8 hexadezimale Zeichen Zeit wann das alte Passwort gesetzt wurde
4 hexadezimale Zeichen Länge des alten Passwortes
Zeichen String altes Passwort

Hinweis: 3

 Die Passwortrichtlinie wird im exportierten Datensatz wie ein Textstring 'ffffnnnllluuudddsss' in hexadezimalem Format abgelegt, z.B. 'F000010004002005003'. Insgesamt sind 6 Hexadezimalwerte enthalten. Der erste Wert ist 4-stelling, alle andere sind 3-stellig.

'ffffnnnllluuudddsss' bedeutet im Detail:

  • ffff : 4 Hexadezimalzeichen als Flag für
    • 0x8000: verwende kleingeschriebene Buchstaben
    • 0x4000: verwende grossgeschriebene Buchstaben
    • 0x2000: verwende numerische Zeichen
    • 0x1000: verwende Sonderzeichen
    • 0x0800: verwende nur Hexadezimalzeichen (0-9,a-f) (wenn gesetzt, kann kein anderes Bit gesetzt sein)
    • 0x0400: verwende ein leicht zu lesendes Passwort (EasyVision)
    • 0x0200: verwende ein aussprechbares Passwort
    • 0x01FF: nicht verwendet
  • nnn  : 3 Hexadezimalzeichen : gesamte Passwortlänge
  • lll  : 3 Hexadezimalzeichen : minimum Anzahl kleingeschriebene Buchstaben im Passwort
  • uuu  : 3 Hexadezimalzeichen : minimum Anzahl grossgeschriebene Buchstaben im Passwort
  • ddd  : 3 Hexadezimalzeichen : minimum Anzahl numerische Zeichen im Passwort
  • sss  : 3 Hexadezimalzeichen : minimum Anzahl Sonderzeichen im Passwort

Die Passwortrichtlinie 'F000010004002005003' gedeutet dann

  • F000 : verwende kleingeschriebene, grossgeschriebene, numerische und Sonderzeichen
  • 010  : verwende mindestens 16 (dez) Zeichen
  • 004  : verwende mindestens 4 kleingeschriebene Buchstaben
  • 002  : verwende mindestens 2 grossgeschriebene Buchstaben
  • 005  : verwende mindestens 5 numerische Zeichen
  • 003  : verwende mindestens 3 Sonderzeichen

Hinweis: 4

Bei der Abspeicherung der Doppel Klick Aktion kommen folgende Werte vor:
  • -1: Standard
  • 0:  Passwort in die Zwischenablage kopieren
  • 1:  Zeige/bearbeite ausgewählten Eintrag
  • 2:  Autom. Eingabe
  • 3:  Gehe zu URL
  • 4:  Notizen in die Zwischenablage kopieren
  • 5:  Benutzername in die Zwischenablage kopieren
  • 6:  Passwort in die Zwischenablage kopieren, minimieren
  • 7:  Gehe zu URL + Autom. Eingabe
  • 8:  Kommando ausführen
  • 9:  Email senden
